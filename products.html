SwitchOn - An EFT Switch

OGS has been in payment industry, every step of the way leading the industry with superior solutions and leveraging high-performance technology to meet the increasing demands and exceed the expectations of the EFT processing business. In order to compete effectively in the global payments market, financial institutions require scalable, reliable, and agile payment systems. OGS SwitchOn allow companies to deliver traditional and innovative payment services with greater efficiency and at lower cost, thereby improving their competitive position in the market.

Debit/Credit Processing - Acquiring & Issuing Switch SwitchOn is a PA DSS v3.1 certified, flexible, robust and multi-channel payment switch for acquiring and routing of authorization transactions from multiple payment channels like ATM, POS, Kiosk, Mobile, and Internet (Payment gateways), into many destination hosts and networks. SwitchOn integrates with point of sale solutions, data warehouses, credit application software, order management systems and custom business management solutions to ensure merchants within our closed-loop payment networks can authorize and settle customer transactions.

Benefits of SwitchOn:-

  • Support multi-channel switching
  • Quick turnaround time for new channel creation; flexible routing and switching rules with event driven alerts
  • Comprehensive switch business functions with multi-channels, multi-hosts and lower total cost of ownership
  • High availability with 24 hours a day, 7 days a week uninterrupted authorization service with Object-Oriented and Service Oriented Architecture (SOA).

Prepaid card solution: OGS leverages its broad EFT payment expertise and assets to make it simple and easy for financial institutions to define and deploy their own differentiated EFT growth strategy. OGS Prepaid Card solution wish to provide pre-paid and/or stored value cards as an in-house managed service. In this scenario, the balance of the cards would be established at a predetermined value and decremented upon usage unlike an ATM or debit card whose card limits would normally be refreshed upon each new business day. Cards could also be established where funds could be reloaded to the card.

Transaction Simulator modules

Transaction Simulator modules( TSM) SimulatOn is a testing and benchmarking tool for applications SwitchOn and its transaction processing subsystem. It is able to archive the raw transactional data into files from device, host and network based inbound/outbound raw data. These archived data files can then be imported into the SimulatOn to run against a test version of the same application on another system. The outputs can then be compared for diagnostics and troubleshooting purposes. All sensitive and confidential data will be maintained with PA DSS compliance/standards.

The main benefit of using it is to provide a safe way to verify that new versions of the application software will operate successfully prior to live implementation. Another benefit is the ability to perform transaction throughput testing to determine hardware sizing, configuration and performance etc.

The following different testing scenarios can be simulated/ tested using SimulatOn

  • EMV Transactions
  • Regression Test Suite
  • Stress Test
  • Host Authorization Simulator
  • ATM terminal Simulator
  • ATM Stress Test Simulator
  • Simulator for Card Number Generation and Track 2 Generation
  • End to end transaction life Cycle(Device-Switch-Host)

Secure On: SSM

As an efficient and cost effective alternative to HSM, OGS has a inbuilt standard Software Security Module (SSM) which is capable of handling various cryptographic operations such as Key Generation, Distribution, PIN verification, translation, card verification etc., in triple DES standards

Handles below listed cryptographic Operations in

  • PIN Generation using HSM
  • PIN Mailer Printing
  • PIN Verification Value (PVV) generation
  • PIN Verification (DUKPT, VISA method etc.)
  • PIN Translation
  • DUKPT support
  • CVV verification

Multi HSM support Transactions originating from an ATM, POS or external network are sent to security module to be initialized and checked. The security module communicates with a third-party device (HSM) which has stored security keys and performs encryption/decryption. The keys are encrypted/decrypted using a function provided by that security device. The process and the function vary depending on the make and model of the security device you are using.

The following are the security devices supported by the system:

  • Thales/Racal
  • Atalla
  • Safenet

Handles security interface between OGS Payment Solution and the Hardware Security Module (HSM) for below listed cryptographic Operations in triple DES standards

  • PIN Generation using HSM
  • PIN Mailer Priniting
  • PIN Verification Value (PVV) generation
  • PIN Verification (DUKPT, VISA method etc.)
  • PIN Translation
  • DUKPT support
  • CVV verification
